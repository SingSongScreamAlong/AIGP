### Durable lessons captured

The session added several template lessons to `project_aigp_state.md` that should outlast individual work:

- Legacy soak "X% failure" can be a timeout artifact masquerading as a distribution (S19w).
- Single-seed RNGs in synthetic detectors convert soak into a single-sample reliability number dressed up as statistics (S19w).
- Per-file `if "mavsdk" not in sys.modules` stubs are a time bomb under multi-file pytest — canonical stub belongs in `conftest.py` (S19x).
- Static unit tests demonstrating a failure aren't evidence the failure matters at system scale; back-check with integration before ranking severity (S19x-f).
- `asyncio.run()` in sync test helpers closes the default loop; use scoped `new_event_loop()` (S19x-f).

## 3. Architecture snapshot

```
[MockKinematic | PX4 SITL | DCL | MockDCL]          ← SimAdapter (ABC)
                    ↓ get_state / get_camera_frame
         ┌──────────┼──────────┐
         ↓                     ↓
[VirtualDetector | YoloPnpDetector | ClassicalGateDetector]
         ↓ List[GateDetection(gate_idx, bearing, range, ...)]
[optional] TaggedDetector(inner, IdentityTagger(gates_ned))   ← S19x
         ↓
[RaceLoop] — _pick_detection → _build_tracker_state → belief.update → navigator.plan → adapter.send_velocity_ned
         ↑ optional PoseFusion (ESKF) drives pos_for_plan instead of adapter truth
```

**Key module inventory:**
- `src/sim/adapter.py` — `SimAdapter` Protocol + `PX4SITLAdapter` + `DCLSimAdapter` (stub) + `MockKinematicAdapter` + `MockDCLAdapter` + `make_adapter(...)`
- `src/vision/detector.py` — `Detector` Protocol + `VirtualDetector` + `YoloPnpDetector` + `ClassicalGateDetector` + `make_detector(...)`
- `src/vision/identity.py` — `IdentityTagger` + `TaggedDetector` + `backproject_ned` (S19x)
- `src/estimation/eskf.py`, `pose_fusion.py` — 15-DOF error-state Kalman filter + gating wrapper
- `src/race_loop.py` — core loop, `_pick_detection` + `_check_gate_pass` + `run(realtime=True|False)`
- `src/race/runner.py` — `RaceRunner.fly(...)` lifecycle + `fly(realtime=...)` forwarding
- `src/race/gate_sequencer.py` — track-agnostic discovery mode
- `gate_belief.py` — `GateBelief` + `BeliefNav` (yaw-fixed, gate-aware fallback, pose_trusted gate)
- `run_race.py` — CLI entrypoint; supports `--backend {px4_sitl,dcl,mock,mock_kinematic,mock_dcl}`, `--detector {virtual,yolo_pnp,classical}`, `--fusion`, `--tag-identities`, `--discovery`
- `soak.py` — sandbox-runnable soak harness (S19w)
- `scripts/day1_dcl.sh` — mechanical day-1 DCL integration sequence (S19v)

**Test suite:** 124/124 under naked `pytest` (~64 s). `conftest.py` installs a canonical mavsdk stub and sets `collect_ignore_glob` for env-only tests. Breakdown in `project_aigp_state.md`.

## 5. What we can't do in the sandbox

Everything load-bearing for the *next* real milestone:

- **Real YOLO wiring.** Needs gate-trained `.pt` weights. PC/GPU-blocked.
- **Distractor-resistant YOLO training.** Needs GPU + data-augmentation pipeline. The VQ2 regime (3D scanned environments with visual distractors) is where this matters most.
- **PX4 SITL re-validation.** Sandbox has no `mavsdk`, no PX4 simulator. Our 100% soak claim is against `mock_kinematic` (timestep-independent kinematics); corroborating against PX4 at N≥20 is a "run this the day you're back on the PC" item.
- **DCL integration.** SDK not yet released.
- **ESKF retune against real Neros IMU.** Hardware-blocked.

The asymmetric-cost argument for staying ahead of DCL — prefer to have scaffolding in place on drop day — still holds, but the marginal return on more sandbox work is now near zero. The sensible next sessions are gated on external access, not internal effort.

## 7. Risk register (delta from 2026-04-20)

**Removed / downgraded:**
- S18 belief regression — **fixed** (S19a-b).
- "5% soak failure rate mystery" — **explained** (S19w: timeout artifact, not algorithmic).

**Still open (unchanged):**
- **Perception transfer to real images is the largest unknown.** Synthetic training → real FPV footage is where any transfer-learning gap lives. Every robustness claim in this report is conditional on synthetic perception.
- **DCL physics ≠ real hardware.** Physical qualifier in September is the real transfer test; DCL is a rehearsal environment.
- **Solo-dev bandwidth.** A single late-stage bug can cost a week; the robustness of the scaffolding around us (tests, CI, soak) is partly insurance against this.

**New (added this epoch):**
- **Tagger over-rejection under real-YOLO pose noise.** `IdentityTagger` defaults (`accept_radius_m=2.5`, `ambiguity_ratio=1.3`) were tuned against sim-truth pose; real EKF2/ESKF pose error may push back-projected detections outside the radius. Mitigation: `--tag-accept-radius` CLI flag for tuning without code changes. Characterise when real weights + real poses are both available.
